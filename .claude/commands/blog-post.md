# Blog Post Writer

블로그 포스트를 작성하거나 수정합니다. 플랜 파일 기반으로 새 포스트를 작성하거나, 기존 포스트를 이어서 수정할 수 있습니다.

## Arguments

User input: $ARGUMENTS

## Instructions

### Step 1: Arguments 해석

`$ARGUMENTS`를 해석합니다:

| 입력 | 동작 |
|------|------|
| 빈 값 | 아래 사용법 안내를 출력하고 종료 |
| `.md` 파일 경로 | 해당 플랜 파일을 읽고 새 포스트 작성 |
| `src/posts/` 경로 | 기존 포스트 디렉토리의 `index.mdx`를 읽고 수정 모드 진입 |

**빈 값일 때 출력:**

```
/blog-post 사용법:

  /blog-post plan.md              → 플랜 파일 기반으로 새 포스트 작성
  /blog-post src/posts/제목       → 기존 포스트 수정 모드

플랜 파일에 포함해야 할 것:
  - 대상 독자 정의
  - 섹션별 핵심 포인트
  - 사용할 코드 스니펫 목록 (파일경로:라인)
  - 도입/마무리 전략
```

---

### Step 2: 사전 검증

1. **브랜치 확인**: `git branch --show-current` 실행. `post/` 접두사가 아니면 사용자에게 확인.
2. **파일 존재 확인**:
   - 플랜 모드: 플랜 파일이 존재하는지 확인. 없으면 에러 안내 후 종료.
   - 수정 모드: `{경로}/index.mdx`가 존재하는지 확인. 없으면 에러 안내 후 종료.

---

### Step 3: 플랜 품질 체크 (플랜 모드만)

플랜 파일을 읽고 다음 항목이 포함되어 있는지 확인:

- [ ] 대상 독자 정의
- [ ] 섹션별 핵심 포인트 + 예상 분량
- [ ] 사용할 코드 스니펫 목록 (파일 경로:라인)
- [ ] 도입/마무리 전략
- [ ] 비유/메타포 설계 (선택)

**부족한 항목이 있으면** 사용자에게 보완 필요 항목을 안내한다. 사용자가 그대로 진행하겠다고 하면 계속 진행.

---

### Step 4: 참조 파일 병렬 읽기

다음 파일들을 **병렬로** 읽는다:

1. **스타일 레퍼런스**: `src/posts/python GIL/index.mdx` — 실제 포스트의 톤, 구조, 서식 참고
2. **소스 자료**: 플랜에 명시된 소스 파일 경로들
3. **코드 예시**: 플랜에 명시된 `파일:라인` 코드 스니펫들

수정 모드일 때는 기존 `index.mdx`를 읽고 수정할 부분을 사용자에게 확인.

---

### Step 5: 포스트 작성

아래 스타일 가이드를 **반드시** 준수하여 포스트를 작성한다.

#### 5-1. Frontmatter

```mdx
---
title: '{제목}'
desc: '{호기심 유발 문장 — 단순 설명이 아님}'
date: '{YYYY-MM-DD}'
thumbnail: /team-member-1.jpeg
ogImage: /b-tree-2.png
category: '{카테고리}'
open: true
---
```

- `desc`: 호기심을 유발하는 문장. "~에 대해 알아봅니다" 같은 단순 설명은 금지.
- `thumbnail`, `ogImage`: 위 값 고정.
- `open: true` 필수.

#### 5-2. 글 구조

- `## N. 제목` 형식의 **번호 매긴 헤딩**
- 섹션 사이 `---` 구분선
- 전체 흐름: **정의 → 비교 → 실사례 → 의미 부여 → 마무리**
- 마무리에 **실행 가능한 체크리스트** 또는 **긍정적 리프레이밍**

#### 5-3. 서식 규칙

- 비교 개념은 **반드시 테이블**로 정리
- 코드 예시는 **설명 직후** 배치
- `<br />` 태그로 소제목 전 여백 추가 (선택적)
- `<details><summary>` 태그로 부가 설명 접기 가능
- `>`(blockquote)로 부연 설명/인용
- **이모지 사용 금지**
- 다이어그램은 **bash 태그**로 선언 (CLAUDE.md 규칙)

```mdx
``` bash
Client → Server → DB
```
```

#### 5-4. 어투

- **평서체** ("~다", "~이다") — 경어체 사용 금지
- 전환 표현: "그럼", "여기서 중요한 건", "반면", "핵심은"
- **과도한 쉼표 사용 금지**
- 자연스러운 흐름을 유지하되 장황하지 않게

#### 5-5. 파일 경로

- 포스트 디렉토리: `src/posts/{제목}/`
- 포스트 파일: `src/posts/{제목}/index.mdx`
- 디렉토리가 없으면 생성

---

### Step 6: 빌드 검증

1. `npm run build` 실행
2. 에러 발생 시 확인 사항:
   - MDX 문법 오류 (닫히지 않은 태그, 잘못된 JSX)
   - 코드 블록 언어 태그 오류
   - 테이블 렌더링 깨짐
3. 에러가 있으면 수정 후 재빌드

---

### Step 7: 결과 안내

- 생성/수정된 파일 경로
- 포스트 구조 요약 (섹션 목록)
- 빌드 결과 (성공/실패)
- 수정이 필요한 부분이 있으면 안내
