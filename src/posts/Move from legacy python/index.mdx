## 서론
갑자기 마이그레이션을 결정했다기 보다 도저히 알아보기 힘든 코드들이 많았다.
레거시라고 하기엔 빌리를 개발한 메인 개발자 분이 눈 부릅뜨고 회사를 다니고 계셨기에, python3 로 마이그레이션 해보기로 한다.

## 명분
Python 2.7은 2020년 1월부터 공식 지원이 중단되어 보안 패치나 버그 수정이 더 이상 제공되지 않는다. (근거 링크 달것)
개인적인 이유로, 이 프로젝트를 주로 맡던 동료 분이 육아휴직을 하셨는데, 돌아오셨을 때 업그레이드된 파이썬 프로젝트를 선물?로 드리고 싶었다.

## Claude Code 를 활용한 마이그레이션 전략
수많은 의존성과 라이브러리 업데이트.

## 성능 향상
Python 3.8은 여러 측면에서 성능이 개선되었습니다. 딕셔너리 연산이 더 빨라졌고, f-string 포매팅이 기존 % 포매팅이나 .format() 메서드보다 훨씬 빠릅니다. 또한 positional-only 매개변수 지원으로 함수 호출 오버헤드가 줄어들었습니다.
(이런 내용을 실제로 측정해볼 수 있으면 좋겠는데, 데이터 독 모니터링을 사용해봐도 좋겠다.)
